# Agent instructions

## Build commands

All build and Elm-related commands must be run inside the devenv shell:

```
devenv shell -- elm-pages build
devenv shell -- elm-pages dev
devenv shell -- elm make ...
devenv shell -- make site
devenv shell -- make assets
devenv shell -- make blay-compose-all
devenv shell -- cabal build --offline
```

Do **not** use `npx`, bare `elm-pages`, or bare `elm` — they are not on PATH outside devenv.

## Project overview

Elm + elm-pages design guide site for Suomen Palikkaharrastajat ry (Finnish LEGO enthusiast org),
served at logo.palikkaharrastajat.fi. Brand assets (SVGs, PNGs, favicons) and design tokens are
generated by a Haskell pipeline.

Key directories:
- `src/Component/` — Elm UI component library (23 modules)
- `app/Route/` — elm-pages routes: Index, Komponentit, Saavutettavuus, Responsiivisuus, Typografia, Varit
- `src/Brand/` — Haskell brand pipeline and generated Elm tokens
- `layout/` — Hand-editable `.blay` brick layout files (committed; CI reads these)
- `app/` — Haskell executables: `blay-render`, `blay-compose`, `blay-draft`, `blay-animate`, `brand-gen`

## Three-tool logo pipeline

**Tool 1 – `blay-draft`** (local only): SVG → draft `.blay`
```
cabal run --offline blay-draft -- --source source.svg --output layout/head-basic.blay
```

**Tool 2 – `blay-compose`** (run locally, commit outputs): derives all `.blay` variants from masters
- Masters: `layout/head-basic.blay` … `layout/head-laugh.blay` — use yellow `F2CD37` as placeholder
- `make blay-compose-all` regenerates all derived blays; commit the result

**Tool 3 – `blay-render`** (CI): reads committed blays → SVG/PNG/WebP/GIF/favicons/design-tokens
- `make render` or `make assets`
- Do not run `blay-draft` or `blay-compose` in CI

## Makefile key targets

| Target | Description |
|--------|-------------|
| `make build` | Compile all Haskell executables |
| `make blay-compose-all` | Derive all `.blay` files (run locally; commit outputs) |
| `make render` | Render committed blays → `logo/` + `favicon/` + design tokens |
| `make assets` | `render` + copy to `public/` |
| `make site` | Full pipeline: `assets` + `elm-pages build` |

## Styling

Tailwind CSS utility classes only — no hand-written CSS files.
Brand tokens: `text-brand` = `#1A1A2E`, `bg-brand-yellow` = `#F2CD37`, `focus:ring-brand` for focus rings.

## Design tokens

- `src/Brand/DesignData.hs` — single Haskell source of truth
- `src/Brand/Json.hs` → `design-guide.json` (W3C Design Tokens 2025.10, gitignored build artifact)
- `src/Brand/JsonLd.hs` → `design-guide/*.jsonld` (JSON-LD split by section, gitignored)
- `src/Brand/ElmGen.hs` → `src/Brand/Tokens.elm` (generated Elm constants, committed)
- Regenerate by running `cabal run --offline brand-gen`

## Component showcase (Komponentit route)

- `app/Route/Komponentit.elm` uses `buildWithLocalState`
- `Model = { tabs : Dict String Tab }`, `Msg = SelectTab String Tab`
- Each component section: Preview / Elm code / JSON-LD tabs
- Extend `Model` and `Msg` when adding interactive demos

## Favicon generation

Favicons are generated by `blay-render` from `layout/square-smile.blay`.
Output in `favicon/` (11 PNG files + favicon.ico), copied to `public/favicon/` by `make assets`.
`elm-pages.config.mjs` injects `<link>` tags for all sizes.

## Git / commit rules

- Do **not** commit `TODO.md` or any `TODO-*.md` files
- Do not commit `design-guide.json` or `design-guide/` (gitignored build artifacts)
- `.blay` files in `layout/` ARE committed (they are source inputs for CI)
- `src/Brand/Tokens.elm` IS committed (generated but checked in)
